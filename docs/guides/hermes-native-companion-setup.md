# Hermes Native Companion Setup

Hermes Companion is a separate native app. The user should not run a companion app from this repository or paste long-lived app secrets into the native app.

## User Flow

1. Register and start the device service on the Mac that will run Hermes work:

   First sign in to the normal user UI at `/control-plane/user`, open Setup, and generate a one-time device registration token.

   ```bash
   HERMES_COMMAND="/path/to/hermes" go run ./cmd/musubi device register \
     --server http://127.0.0.1:8787 \
     --home ~/.musubi/hermes-device \
     --workspace ws_local \
     --name "Hermes Mac" \
     --registration-token <one-time-token> \
     --start
   ```

2. Open Hermes Companion.
3. Hermes Companion opens Musubi in the browser using loopback PKCE.
4. The user signs in at `/control-plane/user`, selects their registered device, and approves:
   - `hermes.task.create`
   - `hermes.task.cancel`
   - `hermes.task.status`
5. Hermes Companion receives a short-lived app session token and sends encrypted task messages through Musubi.

## Admin Flow

For the first implementation there is one admin user. The admin signs in at `/control-plane/admin` with username/password and creates the trusted first-party Hermes Companion app identity. Individual users sign in at `/control-plane/user` and approve or revoke that app for their own devices.

Admin management uses a server-issued HttpOnly SameSite session cookie. App API keys and native app session tokens cannot manage apps, devices, grants, or workspace policy.

## Native Auth Rules

Loopback redirect URIs are intentionally narrow:

- Scheme: `http`
- Host: `127.0.0.1` or `[::1]`
- Port: `49152-65535`
- Path: `/callback`

Musubi rejects `localhost`, external hosts, LAN IPs, wildcard domains, `0.0.0.0`, non-http schemes, arbitrary paths, and ports outside the allowed range.

## Key Boundaries

- Device private key: generated by the device service and kept under the device home. It decrypts device-bound messages.
- Native app private key: generated by Hermes Companion for that install and stored locally by the native app. It decrypts app-bound results.
- App public key: sent to Musubi during native authorization so devices can encrypt results back to the app.
- App session token: short-lived runtime credential from PKCE token exchange.
- App API key: long-lived backend credential. Native Hermes Companion should not use it.

Musubi stores public keys, grants, status, audit metadata, API-key hashes, and session-token hashes. It must not store plaintext Hermes task payloads or private keys.
